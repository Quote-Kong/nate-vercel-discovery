# Quote Kong Analytics & Lead Scoring Strategy

## Overview
When the Project Discovery module is fully merged into the Quote Kong ecosystem, we gain the ability to track the full lifecycle of a lead—from their initial psychological profile and behavioral metrics during intake, directly through to contract signing and payment. 

This document provides programming and architectural notes on how to build predictive Lead Scoring ("The Scorecard") and Conversion Analytics at both the Contractor and System (Quote Kong) levels.

---

## 1. Data Architecture: Closing the Loop
To calculate conversion rates and build a scorecard, the initial discovery submission must be inextricably linked to the final project status in Quote Kong.

### The Decoupled Project Type Challenge
Because the "Lead Magnet Project Types" (`DiscoveryProjectType`) are intentionally decoupled from the core Quote Kong `ProjectType` table, a direct mapping is required. Contractors use marketing-friendly terms for the intake form (e.g., "Master Bath Oasis"), which won't directly match Quote Kong's standardized, production-focused `ProjectType` templates.

**The Solution: Mapping via CompanyClass**
When a contractor creates a new `DiscoveryProjectType` for their public form, the UI must force them to map it to one or more core Quote Kong `ProjectTypes`.

1.  **Filter by CompanyClass:** The list of core `ProjectTypes` presented to the contractor during mapping should be filtered by their assigned `CompanyClass` (e.g., Roofers shouldn't see Kitchen Remodel core types).
2.  **Mapping Table:** Introduce a join table to handle the one-to-many relationship between a marketing project type and the core production types:

    **`DiscoveryProjectMapping` Table:**
    *   `MappingID` (PK)
    *   `DiscoveryProjectTypeID` (FK - The public Lead Magnet type)
    *   `ProjectTypeID` (FK - The core Quote Kong type)
    *   `CompanyID`
    *   `AddedBy`, `AddedOn`, `ChangedBy`, `ChangedOn`

**Required Schema Updates (Main QK DB):**
When a lead is converted to a paid project, the Quote Kong global `Project` table must reference the `SubmissionID` generated by the public intake form.

*   `Project.SourceSubmissionID` (FK to `Submissions.SubmissionID`)
*   `Project.Status` (e.g., `Lead`, `Estimating`, `Contract Sent`, `Won/Paid`, `Lost`)

This allows us to run `JOIN` queries comparing the "Raw JSON Payload" or parsed Discovery tables against `Project.Status = 'Won/Paid'`, and bridging to standard types via the `DiscoveryProjectMapping` table.

## 2. Contractor-Level Reporting
Contractors need to know which of their marketing efforts and incoming project types actually turn into revenue.

**Key Metrics to Build:**
*   **Conversion Rate by Project Type:** 
    `COUNT(Won Projects) / COUNT(Total Submissions) GROUP BY DiscoveryProjectTypeID`
*   **Conversion Rate by Budget Tier:** Do "ballpark cost" seekers convert better or worse than "fixed cost" seekers?
*   **Average Time to Close:** Days from _Submission CreatedAt_ to _Project Status = Won_.

**Dashboard Implementation Note:**
Use charting libraries like `Recharts` to build a "Lead Funnel" visualization on the Contractor dashboard, showing drop-offs at each stage.

## 3. The Lead Scorecard (Predictive Indicators)
We have hidden behavioral metrics in the `Submissions.PayloadJSON` that are strong indicators of a homeowner's intent and readiness to buy. We can build a weighted algorithm to assign a "Hot Lead Score" (0-100) as soon as the email hits the contractor's inbox.

**(+) Positive Indicators (Increase Score):**
*   **High Engagement:** `metrics.testimonialsViewed.length > 2` and `timeSpentPerTestimonial > 15s`. (They care about trust and quality).
*   **High Picture Uploads:** `clientPhotos.length > 2`. (They have spent time planning and visualizing).
*   **Timeline = "ASAP" or "1 to 3 months":** Implies urgency.
*   **Detailed Notes:** Length of `additionalNotes` string > 50 characters.

**(-) Negative Indicators / Tire Kickers (Decrease Score):**
*   **Hesitance/Confusion:** `priorityChangesCount > 5`. (They don't know what they want and keep changing their mind).
*   **Speed-Clicker:** `metrics.timeSpentPerStep < 5s` average. (Didn't read the prompts, just clicking through to see what happens).
*   **Unrealistic Expectations:** Budget is exceptionally low for a "Whole Home" or "Home Addition" project.

**Implementation Logic:**
Create a backend service worker (or cron job) that parses incoming JSON payloads through a scoring function and saves a `LeadScore` integer directly to the `User` or `ClientProperty` table. Show this score via a badge (e.g., 🔥 95/100) next to the lead's name in Quote Kong.

## 4. Quote Kong System-Level Reporting (Global Data)
As the platform owner, observing aggregate data across ALL contractors provides immense value. You can use this data for marketing Quote Kong, advising contractors, or monetizing lead intelligence.

**Solving the Custom Naming Challenge via Mapping:**
Because contractors name their public `DiscoveryProjectType` whatever they want, aggregating data purely on the Lead Magnet name is impossible. However, because we instituted the `DiscoveryProjectMapping` table (linking custom public names back to standardized `ProjectType` records assigned via `CompanyClass`), the Quote Kong system can perfectly aggregate conversion data. The system groups conversions by the standardized `ProjectTypeID`, entirely bypassing the arbitrary marketing names.

**Global Analytics to Build:**
*   **Macro Conversion Rates:** Which standardized `ProjectTypes` (via the mapping table) close the most consistently nationwide?
*   **Regional Trends:** Group leads by `State/Province` to see where Kitchens vs. Basements are trending.
*   **Algorithm Training:** By anonymizing the data, you can take ALL "Won" projects, look back at their original `requirementsScales` and `painPoints` stored in the `Submissions` table, and use Machine Learning (like Logistic Regression) to automatically adjust the weights of the Lead Scorecard. Let the data tell YOU what behaviors result in a signed contract.

**SysAdmin Dashboard Notes:**
Create an Admin-only dashboard tab using `Recharts` and `D3` to visualize global trends. Ensure queries rely on the mapped `ProjectTypeID` for standardization, and keep everything aggregated and anonymized (excluding `CompanyID` and PII) to protect contractor privacy.
